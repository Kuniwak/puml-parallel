# 案1の証明課題に関する規模と有限性の懸念

## 概要

`PLAN.md` の案1では、制御レベルの正規化・積・witness 探索から
Guard / Post を参照する有限個の証明課題（VC）を生成することを想定している。

しかし、この有限性と規模には追加の前提が必要である。

- 有限の制御 LTS と帰納的不変条件を前提にしても、VC 数は Spec の状態数に対して最悪指数的になる。
- 経路を直接展開する方式では、課題数は探索深さに対して指数的になり、ループがあれば一般に無限になる。
- 状態変数を正確に含む記号的正規化では、制御状態が1個でも記号状態数が無限になり得る。

したがって、「有限個の監査可能な VC」という性質は Guard / Post を不透明な述語にするだけでは得られない。
帰納的不変条件、有限抽象化、または探索深さの制限など、別の有限化手段が必要である。

## パラメータ

以下のパラメータを用いる。

- \(S\): Spec の制御状態数
- \(I\): Impl の制御状態数
- \(E_S\): Spec の辺数
- \(E_I\): Impl の辺数
- \(A\): 可視イベント種類数
- \(R\): 到達可能な正規化 Spec と Impl の積状態数
- \(T\): 到達可能な積遷移数

## 帰納的不変条件を用いる場合

Spec の正規化状態は制御状態の部分集合であるため、最大 \(2^S\) 個存在する。
したがって、積状態数と積遷移数には次の上限がある。

\[
R \le I2^S
\]

\[
T \le E_I2^S
\]

初期条件、遷移保存、拒否条件を別々の VC にすると、VC 数は概ね次の規模になる。

\[
N_{VC} \approx 1 + T + R
\]

経路実在性を積遷移ごとに別の VC とするなら、次の規模になる。

\[
N_{VC} \approx 1 + 2T + R
\]

したがって最悪の場合、

\[
N_{VC}=O(2^S(I+E_I))
\]

となる。

複数の VC を1個の巨大な命題にまとめても、全体の論理式量は減らない。VC 数が減る代わりに、
個々の命題が巨大になる。

## O-trace の大きさ

1つの Impl 遷移に対して、対応可能な Spec の辺を選ぶための選言が必要になる。

正規化状態 \(U\) に含まれる Spec 状態から、イベント \(a\) を持つ辺の数を次のように置く。

\[
B_S(U,a)=\sum_{p\in U}d_S(p,a)
\]

O-trace は概ね次の形になる。

\[
前提 \Rightarrow
\bigvee_{k=1}^{B_S(U,a)}
  (Guard_k \land Post_k \land Inv_k)
\]

したがって、1つの O-trace VC の主要な項数は次の規模になる。

\[
O(B_S(U,a))
\]

最悪の場合は \(B_S(U,a)\le E_S\) なので、全 O-trace VC に含まれる述語参照数は次まで増え得る。

\[
O(TE_S)=O(2^SE_IE_S)
\]

通常は同じイベントを持つ出辺数が小さいため、実際の規模はこの上限より小さくなる。

## O-refusal の大きさ

正規化状態 \(U\) の各 Spec 状態と各イベントについて enabledness の包含を検査する場合、
O-refusal は概ね次の形になる。

\[
\bigvee_{p\in U}
  \left(
    Stable_S(p)
    \land
    \bigwedge_{a\in Act}
      (Enabled_S(p,a)\Rightarrow Enabled_I(i,a))
  \right)
\]

トップレベルの項数は次の規模になる。

\[
|U|A \le SA
\]

ただし、正しい enabledness は Guard だけでは決まらない。イベント \(a\) が有効である条件は、
同じイベントを持つ各辺について Guard が成立し、Post を満たす後続 valuation が存在することの論理和である。

\[
Enabled(p,a)
=
\bigvee_{e:p\xrightarrow{a}}
  \left(
    Guard_e(v)
    \land
    \exists v'.Post_e(v,v')
  \right)
\]

そのため、1つの O-refusal に含まれる Guard / Post の参照数は概ね次の規模になる。

\[
O(E_S(U)+|U|\cdot d_I(i))
\]

最悪の場合は次の規模になる。

\[
O(E_S+S\cdot d_I(i))
\]

全積状態について合計すると、最大で概ね次の数の Guard / Post 参照が生成される。

\[
O\left(2^S(IE_S+SE_I)\right)
\]

τ による安定性判定にも、実行可能な τ 後続の不存在を表す同程度の論理式が必要になる。

## O-feasible を経路ごとに生成する場合

長さ \(L\) の経路に対する O-feasible は概ね次の形になる。

\[
\exists v_0,\ldots,v_L.
Init(v_0)\land
\bigwedge_{k=1}^{L}
  (Guard_k(v_{k-1})\land Post_k(v_{k-1},v_k))
\]

したがって、1つの命題の項数は \(O(L)\) となる。各状態に \(V\) 個の変数があるなら、
量化変数数は \(O(LV)\) となる。

一方、平均分岐数を \(b\) とすると、深さ \(L\) までの経路数は次の規模になる。

\[
O(b^L)
\]

全命題量は次の規模になる。

\[
O(Lb^L)
\]

ループが存在すれば経路数は無限になる。そのため、すべての経路について O-feasible を生成する方法は
一般には実装できない。有限化するには、帰納的不変条件による状態・遷移単位の VC へ変換する必要がある。

## 具体的な最大規模の例

次の条件を仮定する。

- Impl 状態数 \(I=10\)
- Impl 辺数 \(E_I=30\)
- 可視イベント種類数 \(A=8\)

制御部分だけでも上限は次の規模になる。

| Spec 状態数 \(S\) | 正規化状態 \(2^S\) | 積状態上限 \(I2^S\) | 積遷移上限 \(E_I2^S\) | VC 数概算 \(R+T\) |
|---:|---:|---:|---:|---:|
| 10 | 1,024 | 10,240 | 30,720 | 約4万 |
| 15 | 32,768 | 327,680 | 983,040 | 約131万 |
| 20 | 1,048,576 | 10,485,760 | 31,457,280 | 約4,194万 |

1つの拒否 VC には最大 \(SA\) 個の enabledness 含意が含まれる。例えば \(S=20\) なら約160個である。
これを積状態全体に掛けると、単純なテキスト展開では数十億項になり得る。

実際には到達可能な部分集合だけを生成するため、多くの場合はこの上限より小さくなる。しかし、
Spec が20状態程度でも構造次第では現実的でなくなる可能性がある。

## 状態数だけでは見積もれない問題

以上の上限は、制御状態の部分集合だけで正規化できるという仮定に基づいている。

状態変数を正確に扱う場合、同じ制御状態でも到達条件が異なる記号状態が生成される。例えば、
次の自己ループを考える。

```text
x' = x + 1
```

この場合、トレースの反復回数によって `x=0`、`x=1`、`x=2`、……という異なる到達条件が現れ得る。
その結果、次の問題が発生する。

- 制御状態数が \(S=1\) でも記号状態数が無限になり得る。
- 最強事後条件の式が反復ごとに成長する。
- 量化消去によって論理式が急激に膨張し得る。
- 制御状態数や辺数だけから有限な上限を導けない。

## 結論

案1の規模と有限性は、実装方式によって次の3段階に分かれる。

1. **有限制御＋ユーザー提供の帰納的不変条件**
   - VC 数は概ね \(O(2^S(I+E_I))\)。
   - 指数的ではあるが、到達可能部分集合が小さければ実装可能。

2. **深さ制限付き経路展開**
   - 深さ \(L\) に対して全命題量は \(O(Lb^L)\)。
   - 有界検査としては実装可能だが、完全な判定にはならない。

3. **正確な記号的正規化または全経路展開**
   - 状態変数とループが存在すると有限性を保証できない。
   - 制御状態数・辺数のみから VC 数の有限上限を示せない。

したがって、`PLAN.md` の「有限個の監査可能な VC」という記述には、
「帰納的不変条件または有限抽象化が与えられる場合」という前提を明記する必要がある。
Guard / Post を不透明な述語として扱うこと自体は、証明課題の有限化や小規模化を保証しない。
